⚠️ Inconsistencias Identificadas

Problemas de consistencia en el diseño de botones

1. Múltiples Estilos para la Misma Función

Problema: Botones "Guardar" usan diferentes clases
Ejemplos: .btn-primary vs .btn-success
Solución: Estandarizar en .btn-primary

2. Colores Inconsistentes

Problema: Rojo usado en múltiples variantes
Ejemplos: .btn-primary, .btn-red, .btn-learning, .btn-area
Solución: Definir paleta de colores clara

3. Tamaños Inconsistentes

Problema: Botones similares con diferentes tamaños
Ejemplos: .btn vs .btn-sm vs .cfg-btn
Solución: Establecer jerarquía de tamaños clara

4. Nomenclatura Confusa

Problema: Nombres de clases poco descriptivos
Ejemplos: .btn-red vs .btn-primary (ambos rojos)
Solución: Renombrar basado en función, no color

💡 Recomendaciones de Mejora

Sugerencias para estandarizar el sistema de botones

1. Estandarizar Colores

Primario: .btn-primary (rojo) - Acciones principales
Secundario: .btn-secondary (gris) - Acciones secundarias
Éxito: .btn-success (verde) - Confirmaciones
Peligro: .btn-danger (rojo oscuro) - Eliminar/detener

2. Simplificar Nomenclatura

Eliminar: .btn-red, .btn-learning, .btn-camera, .btn-area
Mantener: .btn-primary, .btn-secondary, .btn-success, .btn-warning
Agregar: .btn-danger para acciones destructivas

3. Estandarizar Tamaños

Grande: .btn-lg - Para configuraciones principales
Normal: .btn - Para acciones estándar
Pequeño: .btn-sm - Para controles compactos

4. Documentar Sistema

Crear: Guía de estilo para botones
Incluir: Cuándo usar cada variante
Mantener: Consistencia en futuras implementaciones
